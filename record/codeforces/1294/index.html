Problem D. MEX maximizing

Wrong Solution

I came up with the wrong solution during the contest.

During the contest, I have observed that all the moves you are permitted to perform does not move the integer out of the residue class modulo $x$. However, I chose to use std::set to maintain the data. As I wished to take $i$ with minimum $c_ix + i$, where $c_i$ stands for the count of residue class $i$. I inserted the following code to replace the default comparison functor of std::set: std::less<int>.

The problem is, std::set requires its comparison functor always return the same value for the same comparison requests. Since the $c$ array in this functor are mutable, the functor may not return the same value for the same parameters, rendering it useless.

Correct Solution

A smarter solution is to use std::map, this not only fixes the aforementioned issue, but also gives an acceptable complexity. Take the code bellow for reference.

But why does it work?

The map mp is to store the largest one in the residue class that has ever appeared before. The smaller ones can always be constructed. The loop on the 9-th line was to find the first unmarked mp that will be the answer. After all the works, we increase $a_y$ by $x$ so it is “moved” to the next level.

In a word, the map always satisfy the following condition:

  • If $mp_i=1$, then all integers $j$ satisfying $1 \le j \le i$ and $j \equiv i (\operatorname{mod}~x)$ can be constructed using the operations.

Since the $mex$ of the array is monotonically increasing, the $res$ variable is also monotonically increasing. Thus, the time complexity is $O(n \log n)$.

Problem E. Obtain a Permutation

Greedy approach is enough for this problem. For each column, iterate through all its elements. If the element (call it $x$) should be in this column, it must satisfy the following conditions:

  • $ 1 \le x \le nm$
  • $x \equiv j (\operatorname{mod}~m)$

Then, we can calculate how many moves it need to be moved to the correct place, call it $mov$, manage a table containing how many numbers need the same number of moves to be moved to the same place.

For this column, the answer will be $n - \max\{freq-mov\}$. The answer is the sum over all columns.

Problem F. Three Paths on a Tree

Personally, I think this problem is the simplest.

Use two depth-first-searchs to find the diameter, then, cut off all edges involved in the diameter to create a forest. For each node on the diameter, run a dfs on the subtree it is located, and calculate the maximum depth. The depth plus the length of the diameter will be the answer.
