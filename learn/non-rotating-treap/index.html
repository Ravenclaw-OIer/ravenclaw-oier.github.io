Non-Rotating Treaps

Posted on 2020-08-16

Non-rotating treap is a balanced BST. As its name implies, non-rotating treap is a modified version of the Treap data structure. However, unless other balanced BSTs, which employs the “rotating” operation to keep itself balanced, Non-rotating treaps do the work by splitting the tree and merging the parts back wisely so that the time complexity may be controlled at a reasonable level.

Similar to segment trees, before the operation, we pushdown the lazy tag, we update this current node with pushup after each operation.

The merge function expects both of the two given trees are already well-formed (that is, their $key$s are ordered as a BST and their $aux$ values are ordered as a heap). It also expects that all $key$ values of tree $u$ is smaller than those of tree $v$. Based on these assumptions, it simply compares the $aux$ values of the roots of the two trees, and puts them in the right place. If $aux_u>aux_v$, we will let $u$ be the new root node and merge $v$ with its right subtree, otherwise we will let $v$ be the new root node and merge $u$ with its left subtree.

We have already neatly kept the heap property, so we only need to consider the $key$ value now. We only need to decide which of the subtrees will take the root (that is, be splitted out), and continue to work on the other subtree.

Complexity

Since the expected depth of the tree is $O(\log N)$ (based on the randomness of the auxiliary field), the complexity is $O(\log N)$ per operation.

Applications

Non-rotating treaps can solve nearly all problems solvable with segment trees, while it can also elegantly conquer problems that requires operating on moving segments (such as segment reversing, revolving and moving), with great complexity.
